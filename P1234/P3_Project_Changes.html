Changes Made to HTML Files:

We decided to keep our five intended pages. However, we have made minimal changes to the majority of the pages, these are summarized and listed below.

Changes to index.html (Homepage): The slideshow is now updated with fitting deals and discounts. When comparing the intended and the final webpage, we have added more visual buttons for “O’Story” and “Find your restaurant”, which will help the user navigate to the different pages easier. Finally, we decided to add a darker overlay over both the banners of “O’Story” and “Find your restaurant”, considering the intended lighter overlay stood out too much.

Changes to about.html: The intended banner has been replaced with a more fitting image with higher quality. The background images behind the texts have also been replaced with picture collages, which are located below each text, respectively. We added a grey overlay over the collages to make them smoother and fit the general color scheme of the website.

Changes to order.html: We decided to change the images for O’Burger to a Hawaiian Burger, which will correspond with the text in the “O’Story”-section. Soft drink was updated to O’Beverage to fit the O’-name theme. Regarding the receipt, we decided to add a checkbox, which will give the user a student discount if checked. The overall design of the receipt also received some small tweaks, e.g adding the orange lines to separate the receipt from the menu items and the input areas.

Changes to find_restaurant.html: We decided to replace the static image with a live one, using Google Maps API. The overall design on this page also received some small changes:

  • Only restaurant names are in bold text.
  • Removed the horizontal orange border lines.
  • The addresses of each restaurant is now clickable, and will pinpoint location on the map. (see find_restaurant.js in "Changes to intended Javascript files:")

Changes to contact.html: This page didn't undergo many changes. We added an image of Obama and a "How can we help you"-button, which will direct the user down to the FAQ and contact form.

Changes Made to Javascript and CSS:

We decided to add three additional scripts,

Changes to intended Javascript files:

Changes to navbar.js: In P2 we wrote that the O’Burger logo would be “slightly” bigger when scrolling to the top of the page. In the final website, that is an underestimation. The logo is quite enormous. No more changes have been made to the navbar functionality, only style, which will be covered later.

Changes to slideshow.js: The functionality of the slideshow includes everything specified in P2. We also added some lines in the script which make the slideshow stop if the user uses a mobile device (less than 600px, configurable). This to reduce power consumption for mobile devices. The left and right buttons are also hidden when width is less than 600px, but that is done through @media in index.css.

Changes to order.js: All functionalities specified in P2 are implemented, as well as some new ones: The user now has the ability to use discounts to pay less, e.g. student discount (-25%). The discounts are present as checkboxes, and more discounts can easily be added to the page through a single line of code. We added a form to the page where the user sends in information such as name, email and phone number. The form, the submit button, and the receipt are now hidden until the user actually adds items to order. When the submit button is clicked, a function inside of another javascript file called submitImplementation.js is activated. More on that later.

Changes to contact.js: The functionality of contact.js remains the same as in P2, but instead of creating a separate function for the submit button on this page, we did as in order.js, and imported a function from submitImplementation.js and used that instead, in order to effectively reuse code.

Changes to footer.js: The footer functionality remains about the same as in P2. However, a button has been added to the right of the logo. The button reads “Leave a review” and when clicked, redirects the user to a google forms page where they can leave a review for us to see.

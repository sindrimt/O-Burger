Changes Made to HTML Files:

We decided to keep our five intended pages. However, we have mostly made minimal changes , these are summarized and listed below.

Changes to find_restaurant.html: We decided to replace the static image with a live one, using Google Maps API. The overall design on this page also received some small changes:

  • The addresses of each restaurant is now clickable, and will pinpoint location on the map. (see find_restaurant.js in "Changes to intended Javascript files:")

Changes to contact.html: This page didn’t undergo many changes. We added an image of Obama and a “How can we help you”-button that scrolls down to the FAQ questions when clicked.

Changes Made to Javascript and CSS:

We decided to add three additional scripts,

Changes to intended Javascript files:

Changes to navbar.js: In P2 we wrote that the O’Burger logo would be “slightly” bigger when scrolling to the top of the page. In the final website, that is an underestimation. The logo is quite enormous. No more changes have been made to the navbar functionality, only style, which will be covered later. The navbar is smaller for mobile users, and the user has to use two clicks to navigate to any of the pages. This is because the user has to click on the navbar to open the drop down menu, and then click on the desired page.

Changes to slideshow.js: The functionality of the slideshow includes everything specified in P2. We also added some lines in the script which make the slideshow stop if the user uses a mobile device (less than 600px, configurable). This to reduce power consumption for mobile devices. The left and right buttons are also hidden when width is less than 600px, but that is done through a media query in index.css.

Changes to order.js: All functionalities specified in P2 are implemented, as well as some new ones: The user now has the ability to use discounts to pay less, e.g. student discount (-25%). The discounts are present as checkboxes, and more discounts can easily be added to the page through a single line of code. We added a form to the page where the user sends in information such as name, email and phone number. We decided not to add the intended labels in the form, but rather use placeholders with their respective values. The form, the submit button, and the receipt are now hidden until the user actually adds items to order. When the submit button is clicked, a function inside of another javascript file called submitImplementation.js is activated. More on that later.

Changes to contact.js: The functionality of contact.js remains the same as in P2, but instead of creating a separate function for the submit button on this page, we did as in order.js, and imported a function from submitImplementation.js and used that instead, in order to effectively reuse code.

Changes to footer.js: The footer functionality remains about the same as in P2. However, a button has been added to the right of the logo. The button reads “Leave a review” and when clicked, redirects the user to a google forms page where they can leave a review for us to see.

Additional JavaScript files:

Added scroll.js: We came up with a functionality that might look modern and appealing for gen z and millennials. Scroll.js lets us easily create images that scroll with a different speed compared to the rest of the content, which creates a cool “background” feel to the images. Theoretically, since we use classes to identify scroll images, there can exist more than one scroll image on each page, but we only have one scroll image for Find Restaurant, O’bout and Contact.

Added submitImplementation.js: This file acts a little different than the others. It does nothing on its own, but rather provides useful functions that other javascript pages can use. It is used by contact.js and order.js for, as the name might suggest, the submit buttons. submitImplementation.js offers functions that reset all the content of the article.content elements on the page (which are present and standardized on almost every page), and fills the void with a custom message and a button which can be clicked to reload the current page. This javascript file was primarily created to avoid copy-pasting very similar code in the order and contact pages, we thought it was better to rather call a single function from this file instead.

Added find_restaurant.js: This script imports google maps to the website. The Implementation of google maps to a website using javascript requires an API key which is obtained by creating a google maps billing account, which we did. Google themselves provides a code snippet to be used in javascript, but it is only a couple of lines (commented in the script). We coded the rest of the script which adds the feature to click a link that centers the maps to a set location. The script is not completely independent just as a script. The table, and the links are created in find_restaurant.html and called in the script. The html file also contains a script tag which google provided.

Changes to the intended CSS-styling:

We decided to make a global.css which applies to all of the pages, and seperate css-files which applies to their respective html-files. This way, we can make changes to every page in the global.css and overwrite or make separate changes to the html-files in their respective css-files.

global.css:

  • Contains the style for our navbar and footer, some buttons, image classes and scroll functions.
  • Added media queries to reduce the font size of the navbar when the user zooms out or uses a large monitor. This way, the text in the footer won’t disappear.
  • Main font: Changed from Signika Negative SC to Signika.
  • Fallback fonts: Verdana, Geneva, Tahoma, sans-serif
  • Links with borders and gray overlays have white fonts, while the links connected to the google API on the ‘find_restaurant.html’ page have an orange font.
  • The orange font has a lighter hue when the cursor is hovered above the link.
  • The background image in the footer is a bit more zoomed in. Removed the white line in the footer, and positioned the facebook and instagram logo beside each other and vertically centered.

about.css

  • Made the font-size of the paragraphs and headings bigger.
  • Added a solid color (HEX: #2f2f2f) background behind the paragraphs.

contact.css

  • The items in the receipt are written on separate lines.
  • There’s an orange border above and below the receipt.

index.css:

  • Changed the pictures in the slide-show.
  • Added a gray overlay on the pictures under the slide-show.
  • Added text-boxes on top of the images of obama eating an apple, and the map preview.
  • Made the overlay on the images darker.
  • The buttons on the slideshow are now invisible when it’s a mobile device (reasons behind this was covered in the slideshow.js part).

order.css:

  • The overall design of the receipt also received some small tweaks, e.g adding the orange lines to separate the receipt from the menu items and the input areas.

find_restaurant.css:

  • Decreased the margin between the restaurant table-elements.
  • Only the names of the restaurants have a bold font.
  • Removed the orange borders above and below the tables.
  • The bold font-style is only applied to the restaurant names.
  • Removed the horizontal border lines. The color of these links are orange.

Modularity Requirement:

We achieved modularity by making the navbar and footer in javascript. This allows us to include them on every page. We used a global css-file, but also specific css-files in order to overwrite or make specific changes to their respective html-files.
